Three calls. One session.

The whole API fits on a napkin. Create it, prompt it, stream it.

# 1. Create an agent (runtime + model)
curl -X POST https://aod.ravi.id/agents \
  -H "Authorization: Bearer $AOD_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"refactor-bot","model":"anthropic/claude-sonnet-4-6","runtime":"claude"}'

# 2. Start a session
curl -X POST https://aod.ravi.id/sessions \
  -H "Authorization: Bearer $AOD_KEY" \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"<id>","prompt":"refactor payments.py"}'

# 3. Stream the output (SSE)
curl -N https://aod.ravi.id/sessions/<id>/stream \
  -H "Authorization: Bearer $AOD_KEY"

Powered by Sprites.

Every session runs inside a fresh sandbox provisioned on sprites.dev. You bring your own Sprites token and your own model API keys — stored encrypted and scoped per-user. No shared credentials, no credential pooling.

Open source. Contributors welcome.

Agent on Demand is Apache-2.0 and lives on GitHub. We'd especially love help adding more runtimes and model backends. If your favorite agent CLI isn't supported, open a PR.
