# Fairy API Implementation Plan

## Overview

Build an HTTP API that accepts a runtime (claude/codex/gemini), a prompt, and API keys, then spins up a Sprites microVM, executes the specified AI coding agent, and streams events back to the caller via Server-Sent Events.

## Research Summary

Research conducted by 4-agent team (see `thoughts/research/2026-04-15-sprites-platform-research.md`):
- All three AI CLIs are pre-installed in every Sprite — no setup needed
- WebSocket exec endpoint streams binary frames (1-byte stream ID + payload)
- Python SDK (`sprites-py`) is sync surface / async internals, supports streaming via `cmd.stdout`
- `env` param replaces entire environment (must use wrapper script)
- API keys in WebSocket URL query params (must use wrapper script)
- Pricing: ~$0.01-0.02 per 5-minute agent run

## Current State Analysis

Greenfield project. Empty directory with only `thoughts/research/` from the research phase.

## Desired End State

A running FastAPI server with:
```
POST /run  →  SSE stream of agent events
GET /health  →  200 OK
```

Verify by:
```bash
curl -N -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -d '{"runtime": "claude", "prompt": "write hello world in python", "api_key": "sk-ant-..."}'
# → streams NDJSON events line-by-line until agent completes
```

## What We're NOT Doing

- No authentication on the Fairy API itself (add later)
- No Sprite pooling (create-per-request for v1, pooling is a future optimization)
- No repo cloning / file upload into Sprites
- No persistent storage of run history
- No rate limiting on our API
- No multi-tenant isolation
- No frontend/UI

## Implementation Approach

FastAPI + `sprites-py` + `sse-starlette` for SSE streaming. Single-process for v1. The core flow:

1. Receive POST /run with runtime, prompt, api_key
2. Create a Sprite with a unique name
3. Write a wrapper shell script to the Sprite filesystem (exports API key, runs agent CLI)
4. Execute the script via `sprite.command()` with stdout/stderr piped
5. Bridge the binary output stream into SSE events
6. On completion/error, delete the Sprite and close the SSE stream

#### 2. `fairy/sprites_exec.py`

Core execution engine. Creates Sprite, writes wrapper script, runs agent, yields output chunks, cleans up.

```python
import shlex
import uuid
from collections.abc import Generator
from sprites import SpritesClient, SpriteError

from fairy.config import settings
from fairy.runtimes import RUNTIMES, RuntimeConfig


def _build_wrapper_script(config: RuntimeConfig, api_key: str, prompt: str) -> str:
    """Build a shell script that exports the API key and runs the agent.
    
    This avoids two gotchas:
    1. env= on exec replaces the entire environment (no PATH → binary not found)
    2. env= appears in WebSocket URL query params (API key in logs)
    """
    return f"""#!/bin/bash
set -euo pipefail
export {config.env_var}={shlex.quote(api_key)}
export PROMPT={shlex.quote(prompt)}
cd /home/sprite
exec {config.cmd}
"""


class SpriteRunner:
    def __init__(self):
        self.client = SpritesClient(
            token=settings.sprites_token,
            base_url=settings.sprites_base_url,
        )

    def run(
        self,
        runtime: str,
        prompt: str,
        api_key: str,
        timeout: int | None = None,
    ) -> Generator[bytes, None, int]:
        """Create a Sprite, run an agent, yield stdout chunks, return exit code.
        
        Yields bytes from stdout. Stderr is captured separately.
        Returns the process exit code.
        """
        config = RUNTIMES[runtime]
        name = f"{settings.sprite_name_prefix}-{uuid.uuid4().hex[:12]}"
        timeout = timeout or settings.default_timeout

        sprite = self.client.create_sprite(name)
        try:
            # Write wrapper script to Sprite filesystem
            fs = sprite.filesystem()
            script = _build_wrapper_script(config, api_key, prompt)
            (fs / "run-agent.sh").write_text(script)
            sprite.command("chmod", "+x", "/run-agent.sh").run()

            # Execute with streaming — stdout piped through a BinaryIO bridge
            # that yields chunks to the caller
            stdout_reader, stdout_writer = _make_pipe()
            stderr_buf = bytearray()

            cmd = sprite.command("bash", "/run-agent.sh", timeout=float(timeout))
            cmd.stdout = stdout_writer
            cmd.stderr = _ByteArrayWriter(stderr_buf)

            # cmd.run() blocks, so we need to run it in a thread and yield
            # from the pipe. See stream.py for the threading bridge.
            # For now, return the pieces needed:
            return _RunContext(sprite, name, cmd, stdout_reader, stderr_buf)

        except Exception:
            # Cleanup on setup failure
            try:
                self.client.delete_sprite(name)
            except SpriteError:
                pass
            raise

    def cleanup(self, sprite_name: str):
        """Delete a Sprite by name. Best-effort, swallows errors."""
        try:
            self.client.delete_sprite(sprite_name)
        except SpriteError:
            pass
```

**Design note**: The sprites-py SDK's `cmd.run()` is blocking (sync surface, async internals). To stream output from a blocking call into an async SSE response, we need a threading bridge. Phase 3 builds this.

The actual implementation will be simpler than the above skeleton suggests — we may find that setting `cmd.stdout` to a writable pipe and reading from the other end in an async generator is cleaner. The exact pattern depends on how `sprites-py` writes to the BinaryIO. We'll refine in implementation.

## Phase 3: Streaming Bridge

### Overview
Bridge the sprites-py blocking `cmd.run()` (which writes to a BinaryIO) into an async generator that FastAPI's SSE response can consume. The key challenge: `cmd.run()` blocks a thread while writing to `cmd.stdout`, and we need to yield those chunks asynchronously.

### Changes Required:

#### 1. `fairy/stream.py`

Threading bridge: runs `cmd.run()` in a background thread, pipes stdout through a `queue.Queue`, and yields chunks from an async generator.

```python
import asyncio
import io
import json
import queue
import threading
from collections.abc import AsyncGenerator
from dataclasses import dataclass
from sprites import Sprite, ExitError

from fairy.runtimes import RuntimeConfig


_SENTINEL = object()  # signals end-of-stream


class QueueWriter(io.RawIOBase):
    """A writable BinaryIO that puts chunks into a queue.
    
    Assigned to cmd.stdout so sprites-py writes here in real-time.
    """
    def __init__(self, q: queue.Queue):
        self._queue = q

    def writable(self) -> bool:
        return True

    def write(self, b: bytes | bytearray) -> int:
        self._queue.put(bytes(b))
        return len(b)


@dataclass
class RunResult:
    exit_code: int
    stderr: bytes


async def stream_agent_output(
    sprite: Sprite,
    runtime_config: RuntimeConfig,
    timeout: float,
) -> AsyncGenerator[str, None]:
    """Run agent in a thread, yield SSE event strings as output arrives.
    
    Each yielded string is a JSON line suitable for SSE data field:
    - {"type": "output", "stream": "stdout", "data": "..."}
    - {"type": "output", "stream": "stderr", "data": "..."}
    - {"type": "exit", "code": 0}
    - {"type": "error", "message": "..."}
    """
    stdout_q: queue.Queue = queue.Queue(maxsize=1000)
    stderr_q: queue.Queue = queue.Queue(maxsize=1000)
    result_holder: list[RunResult | Exception] = []

    def _run_in_thread():
        try:
            cmd = sprite.command("bash", "/run-agent.sh", timeout=timeout)
            cmd.stdout = QueueWriter(stdout_q)
            cmd.stderr = QueueWriter(stderr_q)
            cmd.run()
            result_holder.append(RunResult(exit_code=0, stderr=b""))
        except ExitError as e:
            result_holder.append(RunResult(exit_code=e.exit_code, stderr=e.stderr or b""))
        except Exception as e:
            result_holder.append(e)
        finally:
            stdout_q.put(_SENTINEL)
            stderr_q.put(_SENTINEL)

    thread = threading.Thread(target=_run_in_thread, daemon=True)
    thread.start()

    loop = asyncio.get_event_loop()
    stdout_done = False
    stderr_done = False

    while not (stdout_done and stderr_done):
        # Poll both queues without blocking the event loop
        for q, stream_name, done_flag_name in [
            (stdout_q, "stdout", "stdout_done"),
            (stderr_q, "stderr", "stderr_done"),
        ]:
            if (stream_name == "stdout" and stdout_done) or (
                stream_name == "stderr" and stderr_done
            ):
                continue
            try:
                chunk = q.get_nowait()
                if chunk is _SENTINEL:
                    if stream_name == "stdout":
                        stdout_done = True
                    else:
                        stderr_done = True
                    continue
                yield json.dumps({
                    "type": "output",
                    "stream": stream_name,
                    "data": chunk.decode("utf-8", errors="replace"),
                })
            except queue.Empty:
                pass

        if not (stdout_done and stderr_done):
            await asyncio.sleep(0.01)  # 10ms poll interval

    # Wait for thread to finish
    thread.join(timeout=5.0)

    # Emit final event
    if result_holder:
        result = result_holder[0]
        if isinstance(result, Exception):
            yield json.dumps({"type": "error", "message": str(result)})
        else:
            yield json.dumps({"type": "exit", "code": result.exit_code})
    else:
        yield json.dumps({"type": "error", "message": "Agent thread did not complete"})
```

**Design rationale**: 
- `QueueWriter` is the simplest BinaryIO that captures real-time writes from the sprites-py SDK's WebSocket receive loop.
- Queue-based polling (10ms) is a pragmatic choice. Under load, we could switch to `asyncio.Queue` with `run_in_executor`, but for v1 this is sufficient and keeps the code simple.
- Each chunk from sprites-py corresponds to one WebSocket frame (one stdout/stderr write from the agent process). These are naturally line-oriented when the agent uses `--output-format stream-json`.

### Success Criteria:
- [ ] `QueueWriter` correctly buffers bytes and signals completion
- [ ] `stream_agent_output` yields valid JSON event strings
- [ ] Thread cleans up on completion and on error

## Phase 4: API Endpoint & Server

### Overview
Wire everything together into FastAPI endpoints: POST /run (SSE streaming), GET /health.

### Changes Required:

#### 1. `fairy/api.py`

```python
import json
from fastapi import FastAPI, HTTPException
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel, Field
from sse_starlette.sse import EventSourceResponse

from fairy.config import settings
from fairy.runtimes import RUNTIMES
from fairy.stream import stream_agent_output
from sprites import SpritesClient, SpriteError


app = FastAPI(title="Fairy", description="AI coding agent orchestration API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)

client = SpritesClient(
    token=settings.sprites_token,
    base_url=settings.sprites_base_url,
)


class RunRequest(BaseModel):
    runtime: str = Field(description="AI runtime: claude, codex, or gemini")
    prompt: str = Field(description="The prompt to send to the agent")
    api_key: str = Field(description="API key for the chosen runtime")
    timeout: int = Field(default=600, ge=10, le=3600, description="Max execution time in seconds")


@app.get("/health")
async def health():
    return {"status": "ok"}


@app.post("/run")
async def run_agent(req: RunRequest):
    if req.runtime not in RUNTIMES:
        raise HTTPException(400, f"Unknown runtime: {req.runtime}. Must be one of: {list(RUNTIMES)}")

    config = RUNTIMES[req.runtime]
    name = f"{settings.sprite_name_prefix}-{__import__('uuid').uuid4().hex[:12]}"

    # Create Sprite
    try:
        sprite = client.create_sprite(name)
    except SpriteError as e:
        raise HTTPException(502, f"Failed to create Sprite: {e}")

    # Write wrapper script
    try:
        from fairy.sprites_exec import _build_wrapper_script
        fs = sprite.filesystem()
        script = _build_wrapper_script(config, req.api_key, req.prompt)
        (fs / "run-agent.sh").write_text(script)
        sprite.command("chmod", "+x", "/run-agent.sh").run()
    except SpriteError as e:
        _cleanup(name)
        raise HTTPException(502, f"Failed to prepare Sprite: {e}")

    async def event_generator():
        try:
            # Emit start event
            yield json.dumps({"type": "start", "runtime": req.runtime, "sprite": name})

            async for event in stream_agent_output(sprite, config, float(req.timeout)):
                yield event
        finally:
            _cleanup(name)

    return EventSourceResponse(event_generator(), media_type="text/event-stream")


def _cleanup(sprite_name: str):
    try:
        client.delete_sprite(sprite_name)
    except SpriteError:
        pass
```

## Phase 5: Error Handling, Cleanup & Testing

### Overview
Add robust error handling (timeouts, Sprite creation failures, agent crashes), ensure Sprites are always cleaned up, and write integration tests.

### Changes Required:

#### 1. Error event types to handle:

| Scenario | Event emitted | Sprite cleanup |
|----------|--------------|----------------|
| Sprite creation fails | HTTP 502 | N/A |
| Script write fails | HTTP 502 | Delete sprite |
| Agent exits non-zero | `{"type": "exit", "code": N}` | Delete sprite |
| Agent times out | `{"type": "error", "message": "timeout"}` | Delete sprite |
| Client disconnects mid-stream | N/A (async generator cleanup) | Delete sprite |
| Sprites API rate limited | HTTP 429 with retry-after | N/A |

The `finally` block in `event_generator()` handles cleanup for all streaming cases. Pre-stream failures use explicit cleanup.

## Testing Strategy

### Automated:
- Unit tests: runtime config resolution, wrapper script generation, request validation
- Integration tests (skip by default): full end-to-end with real Sprites + real API keys

### Manual Testing Steps:
1. Start server with `SPRITES_TOKEN=... uv run python -m fairy.main`
2. Run a Claude agent: `curl -N -X POST localhost:8000/run -H 'Content-Type: application/json' -d '{"runtime":"claude","prompt":"write a python hello world","api_key":"..."}'`
3. Verify SSE stream starts with `{"type":"start",...}`
4. Verify agent output events arrive in real-time (not batched)
5. Verify stream ends with `{"type":"exit","code":0}`
6. Verify Sprite is deleted after run (check `sprites list`)
7. Test timeout: set `"timeout": 10` with a long prompt, verify timeout error event
8. Test client disconnect: start a curl, Ctrl+C midway, verify Sprite cleanup

## Performance Considerations

- Sprite creation adds 1-2s latency per request. Acceptable for v1; pooling eliminates it later.
- 10ms polling interval in stream bridge adds minimal latency (agent output is typically hundreds of ms between chunks).
- Single-process FastAPI handles concurrent requests via async; each request spawns one thread for the blocking sprites-py call. At ~50 concurrent runs, consider gunicorn workers.

## Future Optimizations (NOT in this plan)

1. **Sprite pooling** — maintain warm Sprites, assign on request, return on completion
2. **Authentication** — API keys, rate limiting, tenant isolation
3. **Network policies** — lock down Sprites to only the AI provider's domain
4. **Checkpoints** — pre-clone repos, snapshot pre-configured environments
5. **Run history** — persist run metadata, logs, costs to a database
6. **WebSocket endpoint** — bidirectional for interactive agent sessions (stdin support)
